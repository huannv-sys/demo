Trợ giúp sử dụng Mikrotik Dashboard

API Trang chủ Trợ giúp Cấu hình

Giới thiệu

Mikrotik Dashboard là ứng dụng web cho phép bạn giám sát và quản lý các thiết bị Mikrotik từ một giao diện thống nhất. Ứng dụng này giúp bạn dễ dàng theo dõi trạng thái của các router, lưu lượng mạng và các thông số quan trọng khác.

Hướng dẫn sử dụng

1. Trang chủ - Danh sách Router

Trang chủ hiển thị danh sách tất cả các router Mikrotik đã được cấu hình trong hệ thống. Mỗi router được hiển thị trong một thẻ với các thông tin cơ bản:

  • Tên router: Tên đã được đặt cho thiết bị
  • Địa chỉ IP: Địa chỉ IP của router
  • Model: Loại thiết bị Mikrotik
  • Khách hàng: Tên khách hàng sở hữu router
  • Trạng thái: Hiển thị trạng thái hiện tại của router (online/offline/unknown)

Từ trang chủ, bạn có thể nhấp vào nút "Chi tiết" để xem thông tin chi tiết về một router cụ thể.

Mẹo: Nếu bạn không thấy router nào trong danh sách, hãy kiểm tra lại tệp cấu hình .env và đảm bảo rằng các thông tin router đã được cấu hình đúng.

2. Trang chi tiết Router

Trang chi tiết router cung cấp thông tin toàn diện về một router cụ thể, bao gồm:

  • Thông tin cơ bản: Tên, IP, model, khách hàng và trạng thái
  • Thông tin hệ thống: Phiên bản RouterOS, thời gian hoạt động và mức sử dụng CPU
  • Danh sách interface: Tất cả các interface mạng trên router với các thông số như:
    • Tên và loại interface
    • Trạng thái (hoạt động/không hoạt động)
    • Thống kê lưu lượng (dữ liệu đã nhận/gửi và tốc độ hiện tại)

Lưu ý: Nếu router đang offline, thông tin về interface và các thống kê sẽ không được hiển thị.

3. Trang Cấu hình

Trang cấu hình cung cấp mẫu và hướng dẫn để thêm router mới vào hệ thống. Trên trang này bạn có thể:

  • Xem mẫu định dạng cấu hình router trong tệp .env
  • Sử dụng mẫu để nhập thông tin router mới
  • Tạo mã cấu hình để sao chép vào tệp .env

Lưu ý rằng trang này chỉ tạo mã cấu hình, không thực sự thêm router vào hệ thống. Bạn sẽ cần cập nhật tệp .env và khởi động lại máy chủ.

4. Tài liệu API

Mikrotik Dashboard cung cấp API cho phép tích hợp với các hệ thống khác. Bạn có thể truy cập tài liệu API chi tiết bằng cách nhấp vào nút "API" ở trên cùng của trang. Trên trang tài liệu API, bạn có thể:

  • Xem tài liệu chi tiết về từng API endpoint
  • Thử nghiệm trực tiếp các API
  • Xem mẫu dữ liệu phản hồi từ API

Các endpoint API chính bao gồm:

  • /api/routers: Danh sách tất cả router
  • /api/routers/status: Trạng thái của tất cả router
  • /api/interfaces: Liệt kê tất cả interface của router

Cấu hình Router

Để thêm hoặc chỉnh sửa router trong hệ thống, bạn cần cập nhật tệp cấu hình .env với định dạng sau:

ROUTER1_IP=192.168.88.1
ROUTER1_NAME=Mikrotik Router 1
ROUTER1_USER=admin
ROUTER1_PASSWORD=password_của_bạn
ROUTER1_MODEL=hAP lite
ROUTER1_CLIENT=Office

Để thêm router thứ hai, bạn chỉ cần thêm các dòng tương tự với số chỉ mục tăng lên:

ROUTER2_IP=192.168.89.1
ROUTER2_NAME=Mikrotik Router 2
...

Mẹo: Sau khi cập nhật tệp cấu hình, bạn cần khởi động lại máy chủ để áp dụng các thay đổi.

Xử lý sự cố

Router hiển thị là "offline"

  1. Kiểm tra kết nối mạng giữa máy chủ và router
  2. Xác minh rằng địa chỉ IP của router trong tệp cấu hình là chính xác
  3. Đảm bảo rằng tên người dùng và mật khẩu được cấu hình đúng
  4. Kiểm tra xem API của Mikrotik có được bật trên router hay không

Không thể xem thông tin interface

  1. Đảm bảo router đang online
  2. Kiểm tra xem người dùng đã được cấu hình có đủ quyền để truy cập thông tin interface hay không
  3. Xác minh rằng giao thức API của Mikrotik không bị chặn bởi tường lửa

Dữ liệu lưu lượng không chính xác

Dữ liệu lưu lượng được cập nhật theo thời gian thực khi bạn truy cập trang. Nếu bạn cho rằng dữ liệu không chính xác:

  1. Làm mới trang để cập nhật dữ liệu mới nhất
  2. Đảm bảo rằng đồng hồ hệ thống trên router và máy chủ được đồng bộ hóa
  3. Kiểm tra xem có bất kỳ thiết bị trung gian nào có thể đang lưu trữ dữ liệu không
